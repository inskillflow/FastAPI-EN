# Complete Tutorial: Calculator API with FastAPI and Swagger UI

We are going to create a simple calculator API with FastAPI that includes addition, subtraction, multiplication, and division operations, as well as the interactive documentation generated by Swagger UI.

## Create and manage a Python virtual environment called `monfastapi`:

```bash
cd mon_projet
python -m venv monfastapi
monfastapi\Scripts\activate
python -- version
pip install fastapi uvicorn
deactivate
```

These commands:

1. Navigate to your project folder.
2. Create the `monfastapi` virtual environment.
3. Activate the environment.
4. Install FastAPI and Uvicorn.
5. Deactivate the environment.

You can reactivate the environment later by using:

```bash
monfastapi\Scripts\activate
```

#### Step 1: Install FastAPI and Uvicorn

Start by installing FastAPI and Uvicorn. Run the following commands in your terminal:

```bash
pip install fastapi
pip install "uvicorn[standard]"
```

#### Step 2: Create the main application file

Create a file `main.py` and add the following code to define the API endpoints:

```python
from fastapi import FastAPI, HTTPException

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Welcome to the Calculator API"}

@app.get("/add")
def add(a: float, b: float):
    return {"result": a + b}

@app.get("/subtract")
def subtract(a: float, b: float):
    return {"result": a - b}

@app.get("/multiply")
def multiply(a: float, b: float):
    return {"result": a * b}

@app.get("/divide")
def divide(a: float, b: float):
    if b == 0:
        raise HTTPException(status_code=400, detail="Division by zero is not allowed")
    return {"result": a / b}
```

#### Step 3: Run the application

To start your application, use the following command:

```bash
uvicorn main:app --reload
```

The application will be accessible at `http://127.0.0.1:8000`.

#### Step 4: Access Swagger UI

Swagger UI is an interactive web interface that allows you to test your API. To access it:

1. Open your browser.
2. Go to the URL `http://127.0.0.1:8000/docs`.

# [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

#### Step 5: Using Swagger UI

Once on the documentation page, you will see a list of the available endpoints.

1. **Main endpoint**: `GET /`

   * Description: Displays a welcome message.
   * URL: `http://127.0.0.1:8000/`
   * Example:

     * Request: `GET /`
     * Response: `{"message": "Welcome to the Calculator API"}`

2. **Addition**: `GET /add`

   * Description: Adds two numbers.
   * Parameters: `a` (float), `b` (float)
   * URL: `http://127.0.0.1:8000/add?a=10&b=5`
   * Example:

     * Request: `GET /add?a=10&b=5`
     * Response: `{"result": 15.0}`

3. **Subtraction**: `GET /subtract`

   * Description: Subtracts the second number from the first.
   * Parameters: `a` (float), `b` (float)
   * URL: `http://127.0.0.1:8000/subtract?a=10&b=5`
   * Example:

     * Request: `GET /subtract?a=10&b=5`
     * Response: `{"result": 5.0}`

4. **Multiplication**: `GET /multiply`

   * Description: Multiplies two numbers.
   * Parameters: `a` (float), `b` (float)
   * URL: `http://127.0.0.1:8000/multiply?a=10&b=5`
   * Example:

     * Request: `GET /multiply?a=10&b=5`
     * Response: `{"result": 50.0}`

5. **Division**: `GET /divide`

   * Description: Divides the first number by the second. Raises an exception if the divisor is zero.
   * Parameters: `a` (float), `b` (float)
   * URL: `http://127.0.0.1:8000/divide?a=10&b=5`
   * Example:

     * Request: `GET /divide?a=10&b=5`
     * Response: `{"result": 2.0}`

#### Step 6: Test the API with Swagger UI

* To access Swagger, add `/docs`

# [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

To test each endpoint:

1. Click on an endpoint (for example, `/add`) in Swagger UI.
2. Click the "Try it out" button.
3. Enter the required parameter values (for example, `a` and `b` for `/add`).
4. Click "Execute" to send the request and see the response directly in the interface.

### Conclusion

You now have a simple calculator API with FastAPI, and you can use Swagger UI to test and document your endpoints. This API can easily be extended to include additional features according to your needs. For more information, see the official FastAPI documentation.
